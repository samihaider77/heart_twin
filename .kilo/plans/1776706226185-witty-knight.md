# Plan to Fix 307 Redirect Error on Deployment

## Root Cause Analysis

### Primary Issues Identified

1. **Hardcoded API URL in Frontend**
   - File: `frontend/src/lib/api.ts:5`
   - Uses hardcoded URL: `https://heart-twin-960700171922.europe-west1.run.app/api/v1`
   - `.env.production` defines `NEXT_PUBLIC_API_URL` but it's NOT used
   - **Impact**: When deployed on any server other than that specific Cloud Run instance, frontend still tries to call the hardcoded Cloud Run URL, which may respond with 307 redirects (load balancer, service unavailable, or path redirects)

2. **FastAPI Automatic Trailing Slash Redirects**
   - File: `backend/app/main.py:7-11`
   - FastAPI default `redirect_slashes=True` (not explicitly set)
   - Routes defined:
     - `@router.get("/")` â†’ `/api/v1/patients/` (with trailing slash)
     - `@router.get("/{patient_id}")` â†’ `/api/v1/patients/{patient_id}` (no trailing slash)
   - **Impact**: Accessing `/api/v1/patients` (without slash) triggers FastAPI's automatic 307 redirect to `/api/v1/patients/`
   - Frontend calls: `axios.get(\`\${API_BASE_URL}/patients\`)` â†’ requests `/api/v1/patients` (no trailing slash) â†’ gets 307 redirect to `/api/v1/patients/`

3. **Next.js Trailing Slash Configuration**
   - File: `frontend/next.config.ts`
   - No explicit `trailingSlash` setting
   - Next.js generates 308 redirects for page routes with trailing slashes (not API routes, but could affect navigation)

## Implementation Steps

### Step 1: Fix Frontend API URL to Use Environment Variable
**Files to modify:**
- `frontend/src/lib/api.ts`

**Change:**
Replace hardcoded `API_BASE_URL` with environment variable:
```typescript
const API_BASE_URL = process.env.NEXT_PUBLIC_API_URL + '/api/v1';
```
Or better yet, construct full URLs per request to handle cases where env var already includes the path.

**Reason:** Allows frontend to connect to different backend URLs based on deployment environment.

---

### Step 2: Disable FastAPI Trailing Slash Redirects
**Files to modify:**
- `backend/app/main.py`

**Change:**
```python
app = FastAPI(
    title="Cardiac Digital Twin API",
    version="2.0.0",
    description="Backend for real-time cardiac monitoring and AI analysis",
    redirect_slashes=False,  # Add this to prevent 307 redirects
)
```

**Alternative (if redirects desired):** Ensure all route definitions consistently include trailing slashes (e.g., change `@router.get("/")` to `@router.get("")` and verify all other routes have slashes where needed). The `redirect_slashes=False` approach is cleaner.

---

### Step 3: Update `.env.production` for Target Deployment
**Files to modify:**
- `frontend/.env.production`

**Change:** Update `NEXT_PUBLIC_API_URL` to point to the actual deployed backend URL (without `/api/v1` path):
```env
NEXT_PUBLIC_API_URL=https://your-server.com
```
Or if the backend is served on same origin as frontend:
```env
NEXT_PUBLIC_API_URL=
```
Then in code, use relative URLs like `/api/v1/...`.

**Recommendation:** For same-origin deployment (frontend and backend on same domain), use relative paths:
```typescript
const API_BASE_URL = '/api/v1';
```

---

### Step 4: Verify CORS Settings (Secondary Check)
**File:** `backend/app/main.py:23-29`

Current CORS uses `allow_origins=["*"]` which is permissive but may need to be restricted in production. This doesn't cause 307 but should be reviewed after fixing the URL/redirect issues.

---

### Step 5: Ensure Consistent Trailing Slash Usage (Bonus)
If keeping `redirect_slashes=True`, ensure all routes handle both cases properly by defining routes with explicit trailing slashes or ensuring client always uses correct format. However, Step 2 (disable redirect_slashes) already resolves this.

---
